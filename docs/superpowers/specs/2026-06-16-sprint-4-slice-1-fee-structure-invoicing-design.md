# Sprint 4 · Slice 1 — Fee Structure & Invoicing (Design)

- **Date:** 2026-06-16
- **Status:** Approved (brainstorming complete) — ready for implementation plan
- **Part of:** Sprint 4 (Fees & Payments), slice 1 of ~4. The bursar/proprietor-side foundation.
- **Builds on:** SIS (`ClassLevel`, `Term`, `Enrollment` → `Class` → `ClassLevel`, `Student`), tenancy + permission stack, `School.currency` (currency-as-primitive). Mirrors the assessment module's replace-as-unit config + freeze-snapshot patterns.

## Goal

A bursar/proprietor defines per-(class-level, term) fee items, then bulk-generates a frozen
invoice per enrolled student; each student has a balance. This is the foundation every later
Sprint-4 slice (payments, reconciliation, parent pay) builds on.

## Sprint 4 decomposition (context)
1. **Fee structure & invoicing (THIS slice)** — bursar/proprietor side; no payments.
2. **Payments provider + recording + auto-reconcile + receipts** — `core/payments` (mock +
   Paystack), `Payment` model, webhook → mark invoice paid, receipt.
3. **Reconciliation & collections** — bank-CSV matching, overdue, reminders, finance reports.
4. **Parent self-serve pay (or deferred)** — parent user→Guardian→student identity link +
   one-tap pay.

## Scope (locked decisions, slice 1)
1. **Fee structure = per (class-level, term) named line items**, each `name + amountKobo`.
2. **Invoices bulk-generated by the bursar per term**; `InvoiceLine`s **snapshot** the
   fee-item name+amount at generation (frozen bill — later structure edits never mutate
   issued invoices).
3. **Idempotent generation** — re-running never duplicates; refreshes unpaid invoices; leaves
   any invoice with `paidKobo > 0` untouched.
4. **Money = kobo integers**; all display formatted from `School.currency` (never hardcode ₦).
5. **Discounts/waivers deferred** to a later focused slice (the Invoice model can take
   adjustment lines later without disruption).

### Non-goals
- Payments / Paystack / receipts (slice 2); reconciliation / CSV / overdue / reminders /
  reports (slice 3); parent self-serve + identity link (slice 4); discounts/waivers (deferred).

## Architecture

New `apps/api/src/modules/fees/` module. Three tenant-scoped models + a fees service/controller
(structure config + idempotent generation + reads). Web adds a Settings → Fees config panel +
a `/fees` bursar page. Permissions `fees.view` / `fees.manage` are already in the seed catalog
(proprietor auto-grant covers them).

### Data model (tenant-scoped: TENANT_MODELS + RLS FORCE + migration)
- **`FeeItem`** — `id, schoolId, classLevelId, termId, name, amountKobo Int, order Int`;
  relations to School/ClassLevel/Term; `@@unique([classLevelId, termId, name])`;
  `@@index([schoolId, classLevelId, termId])`.
- **`Invoice`** — `id, schoolId, studentId, termId, classLevelId, totalKobo Int,
  paidKobo Int @default(0), issuedAt DateTime @default(now())`; relations to
  School/Student/Term/ClassLevel + `lines InvoiceLine[]`; `@@unique([studentId, termId])`;
  `@@index([schoolId, termId])`. **Balance = totalKobo − paidKobo** (derived, not stored;
  `paidKobo` stays 0 until slice 2).
- **`InvoiceLine`** — `id, schoolId, invoiceId (→Invoice, onDelete Cascade), name,
  amountKobo Int`; `@@index([schoolId, invoiceId])`. A snapshot copied from `FeeItem`.
- Back-relations on School/ClassLevel/Term/Student. Add `"FeeItem"`, `"Invoice"`,
  `"InvoiceLine"` to `TENANT_MODELS`; RLS FORCE migration for all three (mirror `rls_release`).

### Service — `fees.service.ts` (explicit `schoolId` scoping + tenant-IDOR)
**Structure (`fees.manage`):**
- `setFeeItems(classLevelId, termId, items: { name, amountKobo, order }[])` — validate the
  classLevel + term are this tenant's → 404; **replace-as-unit** for that (classLevel, term):
  in one `$transaction`, delete existing `FeeItem`s where `{ schoolId, classLevelId, termId }`
  and create the new set (explicit `schoolId`). Reject negative `amountKobo` / blank name → 400.
- `getFeeItems(classLevelId, termId)` → ordered items.

**Generation (`fees.manage`):**
- `generateInvoices(termId)`: validate term → 404. Load the term's `Enrollment`s (→ each
  student's `classLevelId` via `class.classLevel`). For each student: gather `FeeItem`s for
  `(classLevelId, termId, schoolId)`; in one `$transaction` per student (or batched), upsert
  the `Invoice` by `(studentId, termId)`:
  - if an existing invoice has `paidKobo > 0` → **skip** (don't disturb a paid bill);
  - else delete its `InvoiceLine`s + recreate from the current fee items + set
    `totalKobo = sum(amountKobo)` + `classLevelId`.
  All writes explicit `schoolId`. Returns `{ created, refreshed, skipped }`.
  - Students whose class level has no fee items → an invoice with `totalKobo = 0`, no lines
    (still created, so the bursar sees them; no crash).

**Reads (`fees.view`):**
- `getInvoices(termId, classId?)` → `[{ studentId, name, classLevelName, totalKobo, paidKobo,
  balanceKobo }]` (optionally filtered to a class); tenant-scoped.
- `getInvoice(studentId, termId)` → `{ student, term, classLevelName, lines: [{ name,
  amountKobo }], totalKobo, paidKobo, balanceKobo }` → 404 if none.

### Money formatting
- API returns raw kobo `Int`s + the school `currency`. Web `formatMoney(kobo, currency)` →
  e.g. `₦50,000.00` for NGN (maps `NGN→₦`, fallback to the ISO code + `Intl.NumberFormat`).
  Pure + unit-tested.

## Validation & errors
- Foreign classLevel/term/student → 404 (explicit `schoolId` finds; `Enrollment` gated via
  the class check).
- Negative amount / blank fee-item name → 400.
- `generateInvoices` with no enrollments → `{ created: 0, ... }`, no crash.
- Re-generation is idempotent (unique `(studentId, termId)`); paid invoices skipped.

## Web
- **Settings → Fees** (`fees.manage`): class-level + term selectors → editable fee-item list
  (name + amount inputs, running total), Save (replace-as-unit). Empty/loading states.
- **`/fees`** (`fees.view`, new nav item): term selector → **Generate invoices** button
  (`fees.manage`, confirm) → table of students with total / paid / **balance** (formatted in
  the school currency) → row click → invoice detail (line items). Optional class filter.
- api client: `getFeeItems`/`setFeeItems`/`generateInvoices`/`getInvoices`/`getInvoice` +
  `formatMoney`.

## Testing
- **Unit (web vitest):** `formatMoney` — NGN → ₦ with thousands + 2dp; unknown currency →
  ISO-coded; 0 kobo → ₦0.00.
- **API e2e** (new `fees.e2e-spec.ts`, service-level inside `TenantContext.run`, model on
  `assessment.e2e-spec.ts`): set fee items per class-level+term; generate → per-student totals
  match their class level's items; **idempotent re-run** (no duplicate invoices; refreshes
  unpaid; a manually `paidKobo>0` invoice is skipped/untouched); editing the structure after
  generation leaves issued `InvoiceLine`s unchanged; student with no fee items → 0-total
  invoice; **cross-tenant** (school B → A's term/classLevel/student) → 404; explicit scoping.
- **Browser QA:** configure fees for a class level + term → generate → `/fees` balances render
  in ₦ → drill into a student's line items.

## Dependencies
- SIS models, tenancy + permission stack (`fees.view`/`fees.manage` seeded), `School.currency`.
  No new npm deps; no payment provider yet.

## Out-of-scope future
- Slices 2–4 (payments, reconciliation, parent pay) + discounts/waivers.
